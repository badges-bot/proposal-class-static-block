    ECMAScript class static initialization blocks

    Introduction

    This proposal defines new syntax to perform privleged static initialization of a class.

    See the proposal repository for background material and discussion.

    1 Executable Code and Execution Contexts

    A class static block environment is a Lexical Environment that corresponds to the evaluation of the static block of a class. A class static block environment establishes a new this binding.

    1.1 Class Static Block Environment Records

    A class static block Environment Record is a declarative Environment Record that is used to represent the top-level scope of a class static block and provides a this binding.

    Class Static Block Environment Records have the additional state fields listed in Table 1.

    1.2 Environment Record Operations

    1.2.1 NewClassStaticBlockEnvironment ( F )

    The abstract operation NewClassStaticBlockEnvironment takes argument F. It performs the following steps when called:

    1. Assert: F is an ECMAScript function.
    2. Let env be a new Class Static Block Environment Record containing no bindings.
    3. Set env.[[FunctionObject]] to F.
    4. Set env.[[ThisBindingStatus]] to uninitialized.
    5. Set env.[[HomeObject]] to F.[[HomeObject]].
    6. Set env.[[OuterEnv]] to F.[[Environment]].
    7. Return env.

    2 ECMAScript Language: Expressions

    2.1 Identifiers

    2.1.1 Static Semantics: Early Errors

    BindingIdentifier:Identifier
    • It is a Syntax Error if the code matched by this production is contained in strict mode code and the StringValue of Identifier is "arguments" or "eval".
    • It is a Syntax Error if the code matched by this production is nested, directly or indirectly (but not crossing function or static initialization block boundaries, excluding arrow function boundaries), within a ClassStaticBlock and the StringValue of Identifier is "await".
    LabelIdentifier:Identifier
    • It is a Syntax Error if the code matched by this production is nested, directly or indirectly (but not crossing function or static initialization block boundaries, excluding arrow function boundaries), within a ClassStaticBlock and the StringValue of Identifier is "await".
    IdentifierReference:Identifier
    • It is a Syntax Error if the code matched by this production is nested, directly or indirectly (but not crossing function or static initialization block boundaries, excluding arrow function boundaries), within a ClassStaticBlock and the StringValue of Identifier is "arguments" or "await".

    3 ECMAScript Language: Statements and Declarations

    3.1 The continue Statement

    3.1.1 Static Semantics: Early Errors

    ContinueStatement:continue; ContinueStatement:continueLabelIdentifier;
    • It is a Syntax Error if this ContinueStatement is not nested, directly or indirectly (but not crossing function or static initialization block boundaries), within an IterationStatement.

    3.2 The break Statement

    3.2.1 Static Semantics: Early Errors

    BreakStatement:break;
    • It is a Syntax Error if this BreakStatement is not nested, directly or indirectly (but not crossing function or static initialization block boundaries), within an IterationStatement or a SwitchStatement.

    4 ECMAScript Language: Functions and Classes

    4.1 Class Definitions

    Syntax

    ClassElement[Yield, Await]:MethodDefinition[?Yield, ?Await] staticMethodDefinition[?Yield, ?Await] ClassStaticBlock ; ClassStaticBlock:static{ClassStaticBlockBody} ClassStaticBlockBody:ClassStaticBlockStatementList ClassStaticBlockStatementList:StatementList[~Yield, ~Await, ~Return]opt

    4.1.1 Static Semantics: Early Errors

    ClassBody:ClassElementList

    • It is a Syntax Error if ClassStaticBlockList of ClassElementList contains more than one element.

    ClassStaticBlock:static{ClassStaticBlockBody}

    • It is a Syntax Error if HasDirectSuper of ClassStaticBlock is true.

    4.1.2 Static Semantics: Contains

    ClassStaticBlock:static{ClassStaticBlockBody}
    1. Return false.
     Note
    Static semantic rules that depend upon substructure generally do not look into static initialization blocks.

    4.1.3 Static Semantics: HasDirectSuper

    ClassStaticBlock:static{ClassStaticBlockBody}
    1. Return ClassStaticBlockBody Contains SuperCall.

    4.1.4 Static Semantics: NonConstructorMethodDefinitions

    ClassElementList:ClassElement
    1. If ClassElement is ClassElement:; or ClassElement:ClassStaticBlock , return a new empty List.
    2. If IsStatic of ClassElement is false and PropName of ClassElement is "constructor", return a new empty List.
    3. Return a List containing ClassElement.
     ClassElementList:ClassElementListClassElement
    1. Let list be NonConstructorMethodDefinitions of ClassElementList.
    2. If ClassElement is ClassElement:; or ClassElement:ClassStaticBlock , return list.
    3. If IsStatic of ClassElement is false and PropName of ClassElement is "constructor", return list.
    4. Append ClassElement to the end of list.
    5. Return list.

    4.1.5 Static Semantics: ClassStaticBlockList

    ClassElementList:ClassElement
    1. If ClassElement is not ClassElement:ClassStaticBlock , return a new empty List.
    2. Return a List containing ClassElement.
     ClassElementList:ClassElementListClassElement
    1. Let list be ClassStaticBlockList of ClassElementList.
    2. If ClassElement is not ClassElement:ClassStaticBlock , return list.
    3. Append ClassElement to the end of list.
    4. Return list.

    4.1.6 Static Semantics: PropName

    ClassElement:ClassStaticBlock
    1. Return empty.

    4.1.7 Static Semantics: LexicallyDeclaredNames

    ClassBlockStatementList:[empty]
    1. Return a new empty List.
     ClassBlockStatementList:StatementList
    1. Return the TopLevelLexicallyDeclaredNames of StatementList.

    4.1.8 Static Semantics: VarDeclaredNames

    ClassBlockStatementList:[empty]
    1. Return a new empty List.
     ClassBlockStatementList:StatementList
    1. Return the TopLevelVarDeclaredNames of StatementList.

    4.1.9 Static Semantics: VarScopedDeclarations

    ClassBlockStatementList:[empty]
    1. Return a new empty List.
     ClassBlockStatementList:StatementList
    1. Return the TopLevelVarScopedDeclarations of StatementList.

    4.1.10 Runtime Semantics: ClassDefinitionEvaluation

    Note
    This algorithm is a delta from the Class Fields proposal.

    With parameter className.

    ClassTail:ClassHeritageopt{ClassBodyopt}
    1. Let lex be the LexicalEnvironment of the running execution context.
    2. Let classScope be NewDeclarativeEnvironment(lex).
    3. Let classScopeEnvRec be classScope's EnvironmentRecord.
    4. If className is not undefined, then
      1. Perform classScopeEnvRec.CreateImmutableBinding(className, true).
    5. Let outerPrivateEnvironment be the PrivateNameEnvironment of the running execution context.
    6. Let classPrivateEnvironment be NewDeclarativeEnvironment(outerPrivateEnvironment).
    7. Let classPrivateEnvRec be classPrivateEnvironment's EnvironmentRecord.
    8. If ClassBodyopt is present, then
      1. For each element dn of the PrivateBoundNames of ClassBodyopt,
        1. Perform classPrivateEnvRec.CreateImmutableBinding(dn, true).
    9. If ClassHeritageopt is not present, then
      1. Let protoParent be the intrinsic object %ObjectPrototype%.
      2. Let constructorParent be the intrinsic object %FunctionPrototype%.
    10. Else,
      1. Set the running execution context's LexicalEnvironment to classScope.
      2. NOTE: The running execution context's PrivateNameEnvironment is outerPrivateEnvironment when evaluating ClassHeritage.
      3. Let superclass be the result of evaluating ClassHeritage.
      4. Set the running execution context's LexicalEnvironment to lex.
      5. ReturnIfAbrupt(superclass).
      6. If superclass is null, then
        1. Let protoParent be null.
        2. Let constructorParent be the intrinsic object %FunctionPrototype%.
      7. Else if IsConstructor(superclass) is false, throw a TypeError exception.
      8. Else,
        1. Let protoParent be ? Get(superclass, "prototype").
        2. If Type(protoParent) is neither Object nor Null, throw a TypeError exception.
        3. Let constructorParent be superclass.
    11. Let proto be ObjectCreate(protoParent).
    12. If ClassBodyopt is not present, let constructor be empty.
    13. Else, let constructor be ConstructorMethod of ClassBody.
    14. If constructor is empty, then
      1. If ClassHeritageopt is present and protoParent is not null, then
        1. Let constructor be the result of parsing the source text 
          constructor(... args){ super (...args);}
          using the syntactic grammar with the goal symbol MethodDefinition[~Yield].
      2. Else,
        1. Let constructor be the result of parsing the source text 
          constructor( ){ }
          using the syntactic grammar with the goal symbol MethodDefinition[~Yield].
    15. Set the running execution context's LexicalEnvironment to classScope.
    16. Set the running execution context's PrivateNameEnvironment to classPrivateEnvironment.
    17. Let constructorInfo be the result of performing DefineMethod for constructor with arguments proto and constructorParent as the optional functionPrototype argument.
    18. Assert: constructorInfo is not an abrupt completion.
    19. Let F be constructorInfo.[[Closure]].
    20. If ClassHeritageopt is present and protoParent is not null, then set F.[[ConstructorKind]] to "derived".
    21. Perform MakeConstructor(F, false, proto).
    22. Perform MakeClassConstructor(F).
    23. Perform CreateMethodProperty(proto, "constructor", F).
    24. If ClassBodyopt is not present, let elements be a new empty List.
    25. Else, let elements be NonConstructorMethodDefinitions of ClassBody.
    26. Let fieldRecords be a new empty List.
    27. For each ClassElement e in order from elements
      1. If IsStatic of e is false, then
        1. Let fields be the result of performing ClassElementEvaluation for e with arguments proto and false.
      2. Else,
        1. Let fields be the result of performing ClassElementEvaluation for e with arguments F and false.
      3. If fields is an abrupt completion, then
        1. Set the running execution context's LexicalEnvironment to lex.
        2. Set the running execution context's PrivateNameEnvironment to outerPrivateEnvironment.
        3. Return Completion(fields).
      4. Append to fieldRecords the elements of fields.
    28. Set the running execution context's LexicalEnvironment to lex.
    29. If className is not undefined, then
      1. Perform classScopeEnvRec.InitializeBinding(className, F).
    30. Set the value of F's [[Fields]] internal slot to fieldRecords.
    31. If ClassBodyopt is not present, let staticBlocks be a new empty List.
    32. Else, let staticBlocks be ClassStaticBlockList of ClassBody.
    33. If staticBlocks contains a single element, then
      1. Let block be the first element of staticBlocks.
      2. Let status be the result of performing ClassStaticBlockEvaluation for block with argument F.
      3. If status is an abrupt completion, then
        1. Set the running execution context's LexicalEnvironment to lex.
        2. Set the running execution context's PrivateNameEnvironment to outerPrivateEnvironment.
        3. Return Completion(status).
    34. Set the running execution context's LexicalEnvironment to lex.
    35. Set the running execution context's PrivateNameEnvironment to outerPrivateEnvironment.
    36. Return F.

    4.1.11 Runtime Semantics: ClassStaticBlockEvaluation

    With parameter F.

    ClassElement:ClassStaticBlock
    1. Perform ClassStaticBlockEvaluation of ClassStaticBlock with parameter F.
     ClassStaticBlock:static{ClassStaticBlockBody}
    1. Let outerContext be the running execution context.
    2. Let innerContext be PrepareForClassStaticBlockEvaluation(F).
    3. Assert: innerContext is now the running execution context.
    4. Perform ClassStaticBlockBindThis(F, innerContext).
    5. Let result be ClassStaticBlockEvaluateBody(ClassStaticBlockBody).
    6. Remove innerContext from the execution context stack and restore outerContext as the running execution context.
    7. ReturnIfAbrupt(result).
    8. Return NormalCompletion(undefined).

    4.1.11.1 PrepareForClassStaticBlockEvaluation ( F )

    When the abstract operation PrepareForClassStaticBlockEvaluation is called with function object F, the following steps are taken:

    1. Let outerContext be the running execution context.
    2. Let privateNameEnv be the PrivateNameEnvironment of outerContext.
    3. Let innerContext be a new ECMAScript code execution context.
    4. Set the Function of innerContext to null.
    5. Set the PrivateNameEnvironment of innerContext to privateNameEnv.
    6. Let innerRealm be F.[[Realm]].
    7. Set the Realm of innerContext to innerRealm.
    8. Set the ScriptOrModule of innerContext to F.[[ScriptOrModule]].
    9. Let localEnv be NewClassStaticBlockEnvironment(F).
    10. Set the LexicalEnvironment of innerContext to localEnv.
    11. Set the VariableEnvironment of innerContext to localEnv.
    12. If outerContext is not already suspended, suspend outerContext.
    13. Push innerContext onto the execution context stack; innerContext is now the running execution context.
    14. NOTE: Any exception objects produced after this point are associated with innerRealm.
    15. Return innerContext.

    4.1.11.2 ClassStaticBlockBindThis ( F, innerContext )

    When the abstract operation ClassStaticBlockBindThis is called with function object F and execution context innerContext, the following steps are taken:

    1. Let innerRealm be F.[[Realm]].
    2. Let localEnv be the LexicalEnvironment of innerContext.
    3. Let envRec be localEnv's EnvironmentRecord.
    4. Assert: envRec is a class static block Environment Record.
    5. Assert: The next step never returns an abrupt completion because envRec.[[ThisBindingStatus]] is not initialized.
    6. Perform envRec.BindThisValue(F).

    4.1.11.3 ClassStaticBlockEvaluateBody ( block )

    1. Perform ? ClassStaticBlockDeclarationInstantiation(block).
    2. Return the result of evaluating block.

    4.1.11.4 ClassStaticBlockDeclarationInstantiation ( block )

    1. NOTE: Only a single Environment Record is needed for top-level vars and lexical declarations.
    2. Let env be the LexicalEnvironment of the running execution context.
    3. Let envRec be env's EnvironmentRecord.
    4. Let varNames be the VarDeclaredNames of block.
    5. Let varDeclarations be the VarScopedDeclarations of block.
    6. Let functionsToInitialize be a new empty List.
    7. Let declaredFunctionNames be a new empty List.
    8. For each d in varDeclarations, in reverse list order, do
      1. If d is neither a VariableDeclaration nor a ForBinding nor a BindingIdentifier, then
        1. Assert: d is either a FunctionDeclaration, a GeneratorDeclaration, an AsyncFunctionDeclaration, or an AsyncGeneratorDeclaration.
        2. Let fn be the sole element of the BoundNames of d.
        3. If fn is not an element of declaredFunctionNames, then
          1. Insert fn as the first element of declaredFunctionNames.
          2. NOTE: If there are multiple function declarations for the same name, the last declaration is used.
          3. Insert d as the first element of functionsToInitialize.
    9. For each n in varNames, do
      1. Perform ! envRec.CreateMutableBinding(n, false).
      2. Call envRec.InitializeBinding(n, undefined).
    10. Let lexDeclarations be the LexicallyScopedDeclarations of block.
    11. For each element d in lexDeclarations, do
      1. NOTE: A lexically declared name cannot be the same as a function/generator declaration, formal parameter, or a var name. Lexically declared names are only instantiated here but not initialized.
      2. For each element dn of the BoundNames of d, do
        1. If IsConstantDeclaration of d is true, then
          1. Perform ! envRec.CreateImmutableBinding(dn, true).
        2. Else,
          1. Perform ! envRec.CreateMutableBinding(dn, false).
    12. For each Parse Node f in functionsToInitialize, do
      1. Let fn be the sole element of the BoundNames of f.
      2. Let fo be the result of performing InstantiateFunctionObject for f with argument env.
      3. Perform ! varEnvRec.SetMutableBinding(fn, fo, false).
    13. Return NormalCompletion(empty).
